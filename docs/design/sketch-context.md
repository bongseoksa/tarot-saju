# 점하나 — 디자인 시안 컨텍스트

> 이 문서는 Sketch AI에 전달하기 위한 디자인 컨텍스트 요약본입니다.
> 모든 내용은 확정된 기획/정의 기반으로 작성되었습니다.

---

## 1. 서비스 개요

- **서비스명**: 점하나 (JeomHana)
- **한줄 소개**: AI가 읽어주는 타로 카드. 무료, 무제한.
- **핵심 가치**: 가볍게 한번 해보는 타로점. 무겁지 않은 신비로움.
- **타겟 유저**: 20~30대, 타로/운세에 가벼운 흥미가 있는 일반 사용자
- **플랫폼**: 모바일 웹 우선 (SPA), 이후 Android/iOS 앱 (WebView 래핑)

## 2. 브랜드 아이덴티티

### 브랜드 의미
> "오늘, 점 하나 찍어볼까?"

- 일상의 가벼운 한마디 — 친구에게 말하듯 가볍게 시작하는 점술
- 시작점(點) — 카드 한 장의 터치가 오늘의 방향을 만든다
- 점(占) + 하나(一) — 가장 먼저 떠오르는 점술 서비스

### 브랜드 톤
- 말투: 친구처럼 편한, "~해볼까?"
- 분위기: 무겁지 않은 신비로움
- 포지셔닝: 심각한 점술 X → 가볍게 한번 해보는 점술 O

### 서비스 내 카피
- 앱 소개: "점 하나 찍으면, 오늘이 달라질지도"
- 공유 메시지: "점하나에서 타로 봤는데 소름..."
- 로딩 화면: "당신을 위한 점, 하나 준비 중"
- 결과 화면 CTA: "점 하나 더 찍어볼까?"
- 히스토리 빈 상태: "여기는 아직 비어있어요. 점 하나 찍으러 가볼까요?"
- 오류 모달: "일시적으로 해석을 불러오지 못했어요. 잠시 후 다시 시도해주세요."

## 3. 캐릭터 — 점하나

서비스의 마스코트이자 안내자. 서비스명 = 캐릭터 이름.

### 형태
- 동그란 점 + 눈 두 개 + 작은 마녀 모자 + 별 장식
- 밝고 귀여운 일러스트 스타일 (카드 이미지와 톤 통일)
- 표정 바리에이션으로 상황별 감정 표현

### 등장 위치별 모습

| 위치 | 점하나의 모습 | 문구 |
|---|---|---|
| 로고 | "점하나" 텍스트 + 캐릭터 | — |
| 파비콘/앱 아이콘 | 점하나 얼굴 (단순화) | — |
| 공통 로딩 화면 | 통통 튀는 애니메이션 | (문구 없음) |
| 카드 뽑기 진입 | 두근두근하는 모습 | "어떤 카드가 나올까?" |
| 결과 스트리밍 중 | 해석 중인 모습 | — |
| 히스토리 빈 상태 | 손 흔드는 모습 | "여기는 아직 비어있어요..." |
| 오류 모달 | 미안한 표정 | "미안, 잠시 후 다시 해볼까?" |
| 공유 이미지 | 워터마크 | — |

### 제작 방식
- AI 생성 (Midjourney/DALL-E)
- 앱 아이콘: 1024x1024 원본, 파비콘 ico(16/32) + apple-touch-icon(180) + webmanifest(192/512)

## 4. 디자인 톤 & 무드

### 무드 키워드
밝고 귀여운 / 가볍게 접근 / 카테고리별 분위기 전환

| 요소 | 방향 |
|---|---|
| 전체 톤 | 밝은 베이스 (라이트 모드) — 친근하고 부담 없는 진입 |
| 무게감 | 가벼움, 귀여움 — 점술 앱이 아닌 감성 앱 느낌 |
| 홈 화면 | 밝은 배경 + 컬러풀한 테마 카드 |
| 결과 화면 | 밝은 배경 유지 — 해석 텍스트 가독성 최적 |
| 카드 선택 화면 | 카테고리별 분위기 전환 (파스텔 배경 색감 변화) |

### 레퍼런스
- 토스/당근 같은 밝고 친근한 앱 톤 + 카테고리별 감성 포인트
- 별자리 앱(Co-Star, The Pattern)의 미니멀 감성 참고

### 피해야 할 것
- 금색 테두리, 한자, 부적 이미지 등 전통 점술 클리셰
- 과도한 이펙트 (반짝이, 별 파티클 등)
- 전체 다크 모드 (가벼운 진입 톤과 불일치)

| 직장 | 소프트 블루 `#DBEAFE` | 미디엄 블루 `#3B82F6` |

## 7. 페이지별 화면 설계

### 네비게이션 원칙
- 탭바 없음 — 선형 플로우, 헤더 + 뒤로가기 방식
- 설정 페이지 없음

### 사용자 흐름
```
[1. 홈] → [2. 카드 뽑기] → [3. 광고(오버레이)] → [4. 결과] → [5. 공유/저장]
```

### 7-1. 홈 (`/`)

**구조:**

| 영역 | 배치 |
|---|---|
| 헤더 | 좌: 점하나 로고(캐릭터+텍스트), 우: 히스토리 아이콘 |
| 퀵 진입 | 홈 상단 — "오늘의 타로 한 장 뽑아볼까?" + CTA 버튼 (인기 테마 1개 바로 진입) |
| 카테고리 탭 | 가로 스크롤 필터 (전체/일상/연애/직장/재물/학업/기타) |
| 테마 카드 리스트 | 세로 스크롤, 카드형 (테마 제목 + 한줄 설명 + 카테고리 태그) |
| 푸터 | 개인정보처리방침 · 이용약관 링크 |

**테마 콘텐츠 (11개):**

| 카테고리 | 테마 | 해시태그 |
|---|---|---|
| 일상 | 오늘의 타로 | #오늘 #하루운세 |
| 일상 | 이번 주 타로 | #이번주 #주간운세 |
| 연애 | 나의 연애운 | #연애 #애정운 |
| 연애 | 그 사람의 마음 | #짝사랑 #속마음 |
| 연애 | 새로운 인연 | #인연 #만남 |
| 직장 | 직장에서의 전망 | #직장 #커리어 |
| 직장 | 이직 타이밍 | #이직 #전환 |
| 재물 | 나의 재물운 | #재물 #금전운 |
| 학업 | 시험/학업 결과 | #시험 #합격 |
| 기타 | 지금 필요한 메시지 | #조언 #메시지 |
| 기타 | 이 선택이 맞을까 | #선택 #결정 |

**상태:**
- 기본: 전체 테마 표시
- 카테고리 필터 활성: 선택된 카테고리의 테마만 표시
- 미완료 세션 모달: "확인하지 못한 결과가 있어요" + [이어서 보기] + [닫기] + □ 더 이상 보지 않기

### 7-2. 카드 뽑기 (`/reading/:themeId`)

**구조:**

| 영역 | 배치 |
|---|---|
| 헤더 | 좌: 뒤로가기, 중앙: 테마 제목 |
| 상단 슬롯 | 가로 3칸 — 과거/현재/미래 라벨, 선택된 카드가 슬롯으로 이동 |
| 카드 그리드 | 22장, 4열 배치 (4x5 + 하단 2장 중앙 정렬), **그리드 영역만 세로 스크롤** |
| 하단 버튼 | 3장 선택 완료 시 활성화 — "점 보기", **하단 고정** |

**레이아웃 원칙:**
- 상단 슬롯(3칸) + 하단 버튼은 화면에 고정 (sticky)
- 카드 그리드 영역만 세로 스크롤 — 소형 기기에서도 카드 크기를 충분히 유지
- 카드 크기: 터치 타겟 최소 44x44 확보, 2:3 비율 유지
- Safe Area: iOS 노치 + 홈 인디케이터, 모바일 브라우저 주소창/툴바 고려 (`safe-area-inset` 적용)

**배경:** 카테고리별 파스텔 색감으로 전환 (5번 컬러 팔레트 참조)

**카드 선택 경험 (4단계 애니메이션):**

1. **기대감** — 카드에 손이 가까워지면(호버/터치) 카드가 살짝 떠오름. 미세한 빛남 효과
2. **몰입** — 터치 시 카드가 위로 떠오르며 뒤집힘. 주변 카드가 살짝 어두워짐 (선택 카드에 집중). 짧은 스케일 펄스(1.05x → 1.0x)
3. **설렘** — 카드 앞면 공개 + 부드러운 글로우 + 카드명 아래에서 페이드인
   - 정방향: 카테고리 포인트 색상 글로우
   - 역방향: 카드가 거꾸로(180도) 공개 + 보라 글로우 + "역방향" 라벨 페이드인
4. **완성감** — 카드가 상단 슬롯으로 부드럽게 이동 (스프링 커브) + 가벼운 바운스 + 라벨 활성화

**상태:**
- 선택 전 (0/3): 모든 카드 뒷면, 슬롯 비어있음
- 선택 중 (1/3, 2/3): 선택된 카드는 슬롯에, 나머지 그리드에서 반투명 처리
- 선택 완료 (3/3): 하단 "점 보기" 버튼 활성화
- 뒤집기 중: 추가 탭 무시

### 7-3. 광고 게이트 (오버레이)

- 별도 페이지 아님, 전면 오버레이
- 공통 로딩 화면 (터치/뒤로가기 차단) → 전면 광고(Interstitial) 재생 → 완료 시 결과 이동
- 실패 시 조용히 스킵 → 바로 결과 이동

### 7-4. 결과 (`/result/:resultId`)

**구조:**

| 영역 | 배치 |
|---|---|
| 헤더 | 좌: 뒤로가기(홈), 중앙: 테마 제목 |
| 카드 요약 | 상단 — 가로 3장 나열 (과거/현재/미래 라벨 + 카드 이미지 + 카드명 + 정/역방향) |
| 해석 텍스트 | 카드 요약 아래 — AI 스트리밍 타이핑 영역, 세로 스크롤 |
| 배너 광고 | 하단 영역 (결과 페이지에만 배치, 홈에는 없음) |
| 하단 고정 | 공유 버튼 + "점 하나 더 찍어볼까?" CTA |

**상태:**
- AI 스트리밍 중: 타이핑 효과로 텍스트가 한 글자씩 나타남 (점하나 캐릭터 "해석 중" 모습)
- 완료: 공유 버튼 활성화, 결과 자동 저장
- 실패: 오류 모달 (점하나 미안한 표정) → 홈 이동

**AI 해석 구조:**
- 과거 카드 해석
- 현재 카드 해석
- 미래 카드 해석
- 종합 조언
- 한줄 요약

---

### 7-5. 히스토리 (`/history`)

**구조:**
- 헤더: 좌 뒤로가기 + "히스토리"
- 결과 목록: 최신순, 최대 20건, 각 항목은 테마명 + 날짜 + 카드 3장 썸네일
- 항목 탭 → 결과 페이지 이동
- 빈 상태: 점하나 캐릭터 (손 흔드는 모습) + "여기는 아직 비어있어요. 점 하나 찍으러 가볼까요?" + 홈 이동 버튼

---

### 7-6. 공유 결과 (`/shared/:shareId`)

- 결과 페이지와 동일한 레이아웃 (카드 요약 + 해석 전문)
- 하단 CTA: "나도 점 하나 찍어볼까?" → 홈 이동
- 공유/저장 버튼 없음 (공유받은 사람은 조회만)

---

### 7-7. 공통 요소

#### 공통 로딩 화면
- 점하나 캐릭터 통통 튀는 애니메이션
- 문구 없음, 터치/뒤로가기 차단
- 광고 로드, AI 호출 등 비동기 대기에 공통 사용

#### 오류 모달
- 점하나 미안한 표정
- "일시적으로 해석을 불러오지 못했어요. 잠시 후 다시 시도해주세요."
- [홈으로 돌아가기] 버튼

#### 미완료 세션 모달 (홈 진입 시)
- "확인하지 못한 결과가 있어요"
- 미완료 세션 목록 (1~3건, 테마명 표시)
- [이어서 보기] 버튼
- [닫기] 버튼
- □ 더 이상 보지 않기 체크박스

## 8. 카드 이미지 에셋

### 필요 에셋
- 메이저 아르카나 22장 (앞면) + 카드 뒷면 1장 = 총 23장

### 스타일 가이드
- 방식: AI 생성 (Midjourney/DALL-E)
- 스타일 키워드: flat illustration / cute / pastel / minimal
- 톤: 밝고 따뜻한, 전통 타로 클리셰 배제
- 각 카드의 전통적 상징을 귀여운 일러스트로 재해석
- 22장 일관성 유지를 위해 동일 프롬프트 베이스 + 카드별 변수만 변경
- 카드 뒷면: 점하나 로고/심볼 + 카테고리 무관 공통 디자인

### 규격
- 권장 비율: 2:3
- 포맷: WebP
- 구체적 픽셀 사이즈는 스케치에서 확정
