# 백로그

MVP 이후 개선/확장 항목. 우선순위별로 관리한다.

> 우선순위: P0(MVP 직후 즉시) → P1(유저 반응 확인 후) → P2(서비스 성장 시)

---

## P0 — MVP 직후 즉시

MVP 출시 후 가장 먼저 착수할 항목. 유저 확보/리텐션에 직접 영향.

| # | 항목 | 이유 | 선행 조건 |
|---|---|---|---|
| 0 | Ollama 서빙 인프라 구축 (Cloudflare Tunnel 또는 클라우드) | Edge Function에서 로컬 Ollama에 접근 불가. MVP 출시의 블로커. 로컬 PC + Cloudflare Tunnel 또는 저비용 클라우드 GPU 필요 | - |
| 1 | 회원가입/로그인 (소셜 — 카카오/구글/애플) | 서버 저장, 히스토리 동기화, 유저 식별의 전제 조건. 기술적 한계 아닌 MVP 범위 제한으로 제외 | - |
| 2 | 결과 히스토리 서버 저장 | 로그인 후 localStorage → DB 마이그레이션. 기기 변경/재설치 시 데이터 보존 | #1 |
| 3 | 카카오톡/인스타 스토리 공유 연동 | 바이럴 핵심 채널. 링크 복사만으로는 공유 전환율 낮음 | - |

## P1 — 유저 반응 확인 후

100명 유저 확보 + 피드백 기반으로 착수 여부 판단.

| # | 항목 | 이유 | 선행 조건 |
|---|---|---|---|
| 4 | 행운패스 (광고 제거 구독) | 수익화 핵심. 점신 벤치마킹 이익률 40% | #1 |
| 5 | 인앱 결제 연동 (Capacitor) | 행운패스 결제 수단 | #4, 앱 출시 |
| 6 | 원카드 / 켈틱크로스 등 추가 스프레드 | 콘텐츠 다양화. 유저 요청 시 우선 | - |
| 7 | 마이너 아르카나 56장 추가 | 카드 풀 확대 → 해석 다양성 증가 | - |
| 8 | 설정 페이지 (알림, 테마 변경 등) | 개인화 기능 | #1 |
| 9 | 푸시 알림 | 리텐션. "오늘의 타로 확인해보세요" | #1, 앱 출시 |
| 10 | AI 응답 속도 최적화 (모델 양자화, GPU 활용) | 유저 이탈 방지. 현재 5~30초 예상 | - |
| 11 | AI 실패 시 광고 보상 (다음 1회 광고 스킵) | 광고 시청 후 결과 못 받은 사용자 보상. 수익화/UX 균형 검토 | - |
| 12 | 오프라인 감지 및 전용 UI | Service Worker 또는 navigator.onLine 활용. 오프라인 시 안내 화면 표시. MVP에서는 기존 에러 처리로 커버 | - |
| 13 | AI 동시 요청 대기열 관리 | Ollama 동시 처리 한계 초과 시 큐잉 시스템 도입. MVP에서는 기본 큐잉 + 150초 타임아웃으로 대응 | 트래픽 증가 |

## P2 — 서비스 성장 시

유료 전환 발생 또는 트래픽 증가 시 검토.

| # | 항목 | 이유 | 선행 조건 |
|---|---|---|---|
| 11 | 로컬 PC → 클라우드 GPU 이전 | 동시 접속 한계 해소, 안정성 확보 | 트래픽 증가 |
| 12 | Gemma → 상위 모델 변경 | 해석 품질 고도화 | 프롬프트 최적화 한계 도달 시 |
| 13 | 카드덱 커스터마이징 | 유료 에셋 판매 가능성 | #7 |
| 14 | 연간/월간 시즌 타로 콘텐츠 | 시즌 이벤트로 유입 증가 | - |
| 15 | 커스텀 도메인 연결 | 브랜딩 | 서비스명 확정 완료 |
| 16 | 모니터링/에러 트래킹 도입 | 안정성 | 트래픽 증가 |
| 17 | 결제 상품 다양화 (단건 결제, 프리미엄 해석 등) | 수익 다각화 | #4, #5 |
