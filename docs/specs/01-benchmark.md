## 1. 점신 (jeomsin.co.kr/tarothome)

### 서비스 구조
- 운세/타로/사주 통합 플랫폼 (국내 1위)
- 타로 섹션에 **40개 이상** 테마별 콘텐츠
- 하단 탭: 점신 / 2026년 운세 / 타로 / 상담 / 점신몰

### 사용자 흐름 (실제 조작 확인)
```
타로홈 → 테마 선택 → 카드 3장 선택 → [결과 보기] → 로딩(분석중) + 광고 → 결과
```

#### 상세 흐름 (Chrome DevTools 분석)

**1단계: 타로홈**
- 상단: "매일 찾게 되는 타로카드" → "평범한 일상 속 타로점" (일상 타로 섹션)
  - "오늘의 타로" / "이달의 타로" 2개 퀵 진입 버튼
- 하단: "타로점으로 고민해결!" → "궁금한 타로를 선택해봐요!" (테마 섹션)
  - 카테고리 탭 6개: 전체 / 26년 타로 / 연애/결혼 / 짝사랑/재회 / 취업/학업 / 그 외
  - 각 테마에 해시태그 2~3개 (#시작 #변화 #확장)
- "카드설정" 버튼으로 카드 덱 디자인 변경 가능

**2단계: 카드 선택 화면**
- 안내 문구: "오늘의 기운을 생각하며 카드 3장을 선택해주세요."
- 상단 결과 슬롯 3개가 미리 표시됨 (카테고리별 이름 표시)
  - 오늘의 타로: "애정운" / "재물운" / "학업&취업운" → **3장이 각각 다른 운세 영역에 대응**
- 카드 22장 (메이저 아르카나만) 뒷면 그리드 배치
- 카드 선택 시 뒤집기 애니메이션
- 3장 선택 완료 → "다시 선택" / "결과 보기" 버튼
- 하단에 Google AdSense 배너 광고 (카드 선택 중에도 노출)

**3단계: 로딩(분석중) 화면**
- "선택한 타로를 분석하고 있어요...." + 퍼센트 표시 (0% → 95%)
- 로딩 중 AdSense 광고 2개 표시 (상단 + 하단)
- **가짜 로딩** — 실제로는 정적 텍스트를 서버에서 가져오는 것이지만, 분석 중인 느낌 연출
- 로딩 시간: 약 5~10초 (광고 노출 시간 확보 목적)

**4단계: 결과 화면**
- 3장의 카드 결과가 **탭 형태**로 표시 (애정운 / 재물운 / 학업&취업운)
- 각 카드별 구성:
  - 카드 이름 (한글 + 영문): "바보 THE FOOL"
  - 카드 이미지 (앞면)
  - **한 줄 요약**: "자유로운 영혼을 가진 상대를 곧 만날 수 있는 하루입니다."
  - **상세 해석**: 약 200~300자 분량의 정적 텍스트
  - 해석이 카드 의미 + 테마(오늘의 애정운)에 맞춰 작성됨
- 하단 추천 섹션:
  - "타로가 전하는 특별한 메시지" / "오늘의 인기 타로" → 추가 테마 유도
  - "당신을 위한 맞춤 타로" → 크로스셀링
- 정방향/역방향 구분 없음 (확인됨)

### 수익 모델
| 수익원 | 방식 |
|---|---|
| 광고 (메인) | 카드 선택 중 배너 + 로딩 중 광고 2개 + 결과 하단 배너 |
| 전문 상담 | 24시 상담사 연결, 가입시 무료 쿠폰 |
| 행운패스 | 광고 제거 구독 (이익률 40%) |
| 점신몰 | 부적/굿즈 판매 |

### 핵심 인사이트
- 타로는 완전 무료 (기회 제한 없음), 광고 수익은 **카드 선택 중 + 로딩 중 + 결과 후** 3중 노출
- 해석은 **정적 텍스트** (AI 아님) — 카드+테마 조합별로 사전 작성됨
- **가짜 로딩 화면**이 광고 노출 시간을 자연스럽게 확보하는 핵심 장치
- 오늘의 타로는 3장이 각각 다른 영역(애정/재물/학업)에 대응 — 한 번에 3가지 운세 제공
- 콘텐츠 양이 압도적 (40+ 테마)

## 2. 헬로우봇 (hellobot.co)

### 서비스 구조
- **채팅형 타로/사주** 플랫폼 — 캐릭터와 대화하듯 운세를 봄
- 2017년 페이스북 메신저 "타로챗봇 라마마"에서 시작
- 주력 캐릭터: 라마마(연애 타로), 풀리피(운세 타로), 판밍밍(사주)
- 누적 600만 DL, 월 매출 약 1.2억 원 추산

### 사용자 흐름 (실제 조작 확인)
```
스킬 스토어 → 스킬 선택 → 이름 입력 → [AD 시작!] → 채팅 → 카드 선택 → 해석 (단계별 채팅)
```

#### 상세 흐름 (Chrome DevTools 분석)

**1단계: 스킬 스토어 (메인 페이지)**
- 상단 탭: 친구 / 스킬 / 채팅 / 무료 / 리뷰 / 기획전
- 카테고리 탭 **16개**: 신년운세 / 사주 / 타로결정 / 연애운솔로 / 결혼미래 / 궁합 / 속마음심리 / 이별재회 / 짝사랑썸 / 재물성공직장 / 임신자녀운 / 점성학별자리 / 관상손금꿈해몽 / 스테디셀러 / 반복구매 많은 / 기타
- "타로·결정" 카테고리 클릭 → 하위 카테고리 7개 표시:
  - 솔로 연애운 / 썸·짝사랑 연애운 / 이별·재회운 / 커플 연애운 / 인간관계운 / 진로·합격운 / **기타 타로종합** / 반려동물운
- 기획전 배너: 기획전별 묶음 할인 판매 (수익 극대화 전략)

**2단계: 스킬 상세 페이지** (예: "금전운을 높이려면?" - 무료 스킬)
- 캐릭터 이름: "풀리피"
- 스킬 타이틀: "금전운을 높이려면?"
- 메타 정보:
  - 별점: 4.8 / 이용자 수: 30만+ / 글자 수: 약 350자
- 스킬 소개:
  - 해시태그: "#금전운 #타로 #행운의장소 #올라가라행운행운"
  - "누가 보면 좋을까요?" → 타겟 유저 3가지 체크리스트
  - 소개 텍스트: 구체적 가치 제안
- **미리보기 이미지 3장** — 채팅 UX 스크린샷으로 결과물 미리 확인 가능
- 평가 시스템: 총 84,516개 평가 (이모지 기반)
  - 5가지 반응: 우-우 / 재밌어요 / 도움 돼요 / 힘이 돼요 / **고마워요** (가장 많음)
  - 별점이 아닌 **감정 이모지** 기반 평가 → 사용자 감정 연결 강화
- 추천 연관 스킬 5개 (크로스셀링)
- 가격: "무료" (유료 스킬은 17,000~25,000원대, 정가+할인가 병행)
- 하단 CTA: "시작하기" 버튼

**3단계: 이름 입력**
- 채팅방 진입 전 "제가 뭐라고 부를까요?" → 이름 입력 (최대 20자)
- 약관 동의 내재 (시작과 동시에 동의)
- **이름을 해석에 자연스럽게 삽입** ("테스트, 금전운 높이기 타로를 선택했구나")

**4단계: 광고 게이트 (무료 스킬)**
- 3가지 선택지 제공:
  - **"AD 시작!"** → 광고 시청 후 무료 진행
  - **"300원 광고 스킵하고 볼게!"** → 소액 결제로 광고 없이 진행
  - **"다음에 볼게"** → 취소
- **"그만하기"** 옵션도 항상 노출
- 무료 스킬에서도 수익화 기회를 만드는 전략

**5단계: 채팅형 카드 선택**
- 캐릭터(풀리피)가 대화 메시지로 안내:
  - "좋아! 잘 선택했어"
  - "네 손에 들어올 돈을 생각하며, 느낌이 오는 카드를 한 장 골라봐"
  - 카드 뽑기 안내 GIF 이미지 첨부
- 카드 선택 UI:
  - **Swiper 슬라이더** (좌우 스와이프)로 카드 배치
  - **78장 전체** (메이저+마이너 아르카나) — 점신(22장)과 차별
  - 카드 뒷면 이미지 + 번호 표시 ("tarot card no. 1" ~ "no. 78")
  - 카드 크기: 104x165px, cursor-pointer, 터치 시 위로 튀어오르는 애니메이션
- 원카드 스킬: 카드 1장만 선택

**6단계: 해석 결과 (채팅형)**
- 결과가 **한 번에 전체가 아닌 여러 채팅 메시지로 단계별 전달**:
  - 1st 메시지: "[카드 이미지] 달 카드가 등장했다"
  - 2nd 메시지: 카드 의미와 현재 상황 연결 ("혹시 요즘 기분이 좀 울적했어?")
  - 3rd 메시지: 구체적 분석 ("충동적으로 돈도 많이 썼을 거 같다")
  - 각 메시지 사이 이모지 반응 버튼 제공 (예: "😭")
  - 반응 선택 → 다음 메시지 진행 (사용자 참여 유도)
  - 4th 메시지: 조언 ("감정관리를 잘 해줘야 해")
  - GIF 이미지 삽입으로 감성 보강
- 해석 중간에 "그만하기" 옵션 항상 노출
- **정적 텍스트**이지만 채팅 형태로 타이핑 효과를 줌 → 스트리밍 느낌 유사

### 수익 모델
| 수익원 | 방식 |
|---|---|
| 스킬 단건 결제 | 개별 타로/사주 콘텐츠 구매 (17,000~25,000원) |
| 월 구독 (VIP 클럽) | 26,000원/월, 자동 갱신 |
| 하트 구매 | 인앱 재화, 10% 페이백 |
| 기획전 | 묶음 할인 판매 |
| 무료 스킬 광고 | 리워드 광고 + 광고 스킵 소액 결제 (300원) |

### 핵심 인사이트
- **채팅형 UX가 몰입감의 핵심** — 해석을 단계별로 전달하며 반응 유도
- **이모지 반응 → 다음 메시지** 패턴이 사용자 참여를 자연스럽게 만듦
- 78장 전체 카드 + Swiper 슬라이더 = 카드 선택의 물리적 체험감
- **이름 개인화**: 해석에 사용자 이름을 삽입 ("테스트, 혹시 요즘...")
- 무료 스킬에서도 "광고 스킵 300원" 옵션으로 소액 수익화
- 캐릭터 세계관이 결제를 자연스럽게 유도
- 약점: "유료로 너무 전환되서 잘 안씀" — 무료 콘텐츠 부족

## 3. 우주고양이 보라 (IF Corp.)

### 서비스 구조
- **캐릭터 중심 운세** 앱 — "보라"라는 고양이 캐릭터
- 4명의 유명 사주가 + 카이스트 연구진 협업
- 2023년 10월 출시, 2년 만에 100만 DL, 평점 4.8

### 사용자 흐름
```
앱 진입 → 오늘의 운세 (무료) → 테마 선택 → 카드 선택 → 결과
```
- 오늘의 운세는 무료 제공 (매일 방문 유도)
- 타로: 카드 중 하나를 선택하여 직관적 조언
- Yes or No 카드, 행운카드 등 가벼운 무료 콘텐츠

### UX/UI 특징
- **고양이 캐릭터 "보라"가 중심** — 감성적 브랜딩
- 따뜻한 톤의 손글씨 스타일 메시지
- **일기 기능**: 일기를 작성하면 보라가 답변 (리텐션 장치)
- **편지 기능**: 캐릭터가 사용자에게 편지를 보냄
- 앱 사용 기간 표시 ("보라와 함께한 OO일")
- 직관적 카드 선택 인터페이스

> 보라는 모바일 전용 앱으로 웹 버전이 없어 Chrome DevTools 직접 분석 불가.
> 앱 스토어 스크린샷, 리뷰, 웹 리서치 기반 분석.

### 수익 모델
| 수익원 | 방식 |
|---|---|
| 젤리 (인앱 재화) | 유료 운세 열람에 젤리 소모 (400젤리~) |
| 매일 무료 젤리 | 출석/광고로 소량 지급 (리텐션) |
| 프리미엄 리포트 | 신년운세 등 상세 분석 (50페이지) |
| 인앱 구매 | 젤리 패키지 구매 |

### 핵심 인사이트
- **캐릭터 + 감성이 핵심 경쟁력** — 기능보다 "위로"에 집중
- 일기/편지 기능이 매일 방문 유도 (DAU 장치)
- 무료 콘텐츠 넉넉 → 자연스럽게 유료 전환
- 사용자 리뷰: "화가 나거나 짜증날 때 자주 사용", "캐릭터가 위로해주는 느낌"
- 약점: 젤리 무료 지급량 부족, 운세보관함 편집 불가

## 6. 차별화 전략

### 우리만의 핵심 차별점

| 차별점 | 경쟁사 현황 | 우리의 강점 |
|---|---|---|
| **AI 생성 해석** | 모든 경쟁사가 정적 텍스트 또는 사람 | 매번 다른 해석, 개인화된 문장 |
| **스트리밍 표시** | 헬로우봇은 채팅 단계별 전달(정적), 나머지 없음 | AI가 실시간으로 써내려가는 몰입감 |
| **정/역방향 해석** | 점신은 정방향만, 헬로우봇만 지원 | 해석 깊이 2배 |
| **무료 + 저진입장벽** | 헬로우봇은 유료, 보라는 재화 제한 | 점신 수준 접근성 + AI 품질 |
| **결과 공유** | 대부분 미비 | 카드 이미지 + 해석 요약 → 바이럴 |

### 벤치마킹할 UX 패턴

| 출처 | 패턴 | 적용 방안 |
|---|---|---|
| 점신 | 테마 = 질문, 해시태그 미리보기 | **MVP 적용 완료** |
| 점신 | 매회 광고 게이트 | **MVP 적용 완료** |
| 점신 | 가짜 로딩(분석중) → 광고 노출 시간 확보 | MVP에서 AI 실제 생성 시간이 이 역할을 대체 |
| 점신 | 3장이 각각 다른 운세 영역(애정/재물/학업) | MVP는 과거/현재/미래, Phase 2 검토 |
| 헬로우봇 | 채팅형 단계별 결과 전달 | Phase 2 검토 (AI + 채팅 UI) |
| 헬로우봇 | 이름 개인화 ("OO아, 혹시...") | Phase 2 (프롬프트에 이름 주입) |

| 보라 | 캐릭터가 위로하는 감성 | AI 프롬프트 톤에 반영 |
| coto | 심리학 기반 전문가 포지셔닝 | AI 프롬프트 심리 기법에 이미 반영 |

### AI 실제 생성 시간의 전략적 활용

점신이 "가짜 로딩" 화면으로 광고 노출 시간을 확보하는 것에 착안:
- 우리 서비스는 AI가 실제로 해석을 생성하는 시간(5~30초)이 존재
- 이 대기 시간이 자연스러운 "신비로운 분위기 연출" + 광고 노출 시간이 됨
- 점신의 가짜 로딩 → 우리의 실제 AI 생성 = 사용자 경험이 더 진정성 있음
- 광고를 "결과 보기 전"이 아닌 "AI 생성 중"에 배치하면 대기 지루함을 줄이면서 수익화 가능

## 8. UX/UI 대중성 기준

벤치마킹 대상 4개 서비스에서 공통적으로 발견되는 **대중적 UX/UI 패턴**.

### 필수 패턴 (MVP에 반영)

1. **진입 즉시 콘텐츠 노출** — 홈에서 바로 테마 목록 보여줌 (점신/헬로우봇)
2. **3단계 이내 결과 도달** — 테마 선택 → 카드 뽑기 → 결과 (점신/보라)
3. **카테고리 탭 필터** — 테마를 분류하여 빠른 탐색 (점신 6개, 헬로우봇 16개)
4. **다크 톤 카드 선택 화면** — 신비로운 분위기 연출 (점신)
5. **일일 무료 콘텐츠** — 오늘의 운세/타로로 매일 방문 유도 (전체)
6. **상단 결과 슬롯 미리 표시** — 카드가 들어갈 위치를 미리 보여줌 (점신)

### 권장 패턴 (Phase 2)

7. **캐릭터/브랜딩** — 서비스를 인격화하여 감정적 유대 형성 (보라/헬로우봇)
8. **채팅형 인터랙션** — 대화하듯 진행하여 몰입감 증대 (헬로우봇)
9. **이름 개인화** — 해석에 사용자 이름 삽입 (헬로우봇)
10. **감성적 톤** — "위로", "공감", "따뜻함" (보라)
11. **가격 할인 표시** — 정가 대비 할인가를 함께 노출 (헬로우봇)
12. **이모지 기반 평가** — 별점 대신 감정 반응으로 평가 (헬로우봇)
13. **리텐션 루프** — 일기/편지/출석 보상으로 DAU 유지 (보라)
