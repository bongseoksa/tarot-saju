# 타로 MVP 기능 설계

## 핵심 목표
- 빠른 체험 → 감정 자극 → 공유 유도
- 무제한 무료 이용 + 결과 보기 전 매회 광고 시청

---

## 사용자 흐름

> 벤치마킹(점신) 반영하여 단순화. 상세 분석: [01-benchmark.md](./01-benchmark.md)

```
[1. 홈 (테마 목록)] → [2. 카드 뽑기 (3장)] → [3. 광고 시청] → [4. AI 해석 결과] → [5. 공유/저장]
```

### IA (정보 구조)

```
홈 (/)
  ├── 헤더: 점하나(로고) + 히스토리 아이콘
  ├── 상단: 오늘의 타로 / 이번 주 타로 (퀵 진입)
  ├── 카테고리 탭 필터 (전체/연애/직장/재물/학업/기타)
  ├── 테마 카드 목록
  └── 푸터: 개인정보처리방침 | 이용약관 (노션 링크)

카드 뽑기 (/reading/:themeId)
  ├── 헤더: [← 뒤로] + 테마명
  ├── 상단: 결과 슬롯 3개 (과거/현재/미래)
  ├── 중앙: 카드 22장 그리드
  └── 하단: 다시 선택 / 결과 보기

광고 (오버레이/전면)
  └── 전면 광고 → 완료 시 결과 페이지 이동

결과 (/result/:readingId)
  ├── 헤더: [← 뒤로(홈)] + 공유 아이콘
  ├── 카드 3장 요약 (이미지 + 카드명 + 정/역방향)
  ├── AI 스트리밍 해석 (과거/현재/미래/종합 조언/한줄 요약)
  └── 하단: "점 하나 더 찍어볼까?" (홈 이동)

히스토리 (/history)
  ├── 헤더: [← 뒤로] + 히스토리
  ├── 결과 목록 (최신순, 최대 20건, localStorage)
  ├── 빈 상태: "아직 본 점이 없어요"
  └── 항목 탭 → 결과 페이지 이동
```

### 네비게이션

- 탭바 없음 — 선형 플로우에 탭바 불필요
- 헤더 + 뒤로가기 방식
- 설정 페이지 없음 — 개인정보처리방침/이용약관은 홈 푸터 링크 (노션 페이지)

### 라우팅

| 경로 | 페이지 | 비고 |
|---|---|---|
| `/` | 홈 (테마 목록) | SEO 프리렌더 대상 |
| `/reading/:themeId` | 카드 뽑기 | 동적 라우트 |
| `/result/:resultId` | 결과 (AI 해석) | resultId = 로컬 생성 UUID |
| `/history` | 히스토리 목록 | localStorage 기반 |
| `/shared/:shareId` | 공유 결과 보기 | D-3에서 상세 확정 |
| `*` (캐치올) | → `/` 리다이렉트 | 별도 404 페이지 없음 |

- 광고 게이트: 별도 URL 없음. 카드 뽑기 → 결과 전환 사이 오버레이
- 카테고리 필터: `/?category=love` (선택적, 뒤로가기 시 유지)
- 약관/개인정보: 노션 외부 링크

### 화면별 정의

#### 1. 홈 (테마 목록)
- 헤더: 점하나 로고 + 히스토리 아이콘
- 상단: 일상 타로 섹션 (오늘의 타로, 이번 주 타로)
- 카테고리 탭 필터: 전체 / 연애 / 직장 / 재물 / 학업 / 기타
- 테마 카드 목록 (해시태그 키워드 포함)
- 테마 선택 시 바로 카드 뽑기 진입 (별도 질문 선택 없음)
- 푸터: 개인정보처리방침 | 이용약관

#### 테마 콘텐츠 (테마 = 질문)

| 카테고리 | 테마 (MVP) | 해시태그 |
|---|---|---|
| 일상 | 오늘의 타로 | #오늘 #하루운세 |
| 일상 | 이번 주 타로 | #이번주 #주간운세 |
| 연애 | 나의 연애운 | #연애 #애정운 |
| 연애 | 그 사람의 마음 | #짝사랑 #속마음 |
| 연애 | 새로운 인연 | #인연 #만남 |
| 직장 | 직장에서의 전망 | #직장 #커리어 |
| 직장 | 이직 타이밍 | #이직 #전환 |
| 재물 | 나의 재물운 | #재물 #금전운 |
| 학업 | 시험/학업 결과 | #시험 #합격 |
| 기타 | 지금 필요한 메시지 | #조언 #메시지 |
| 기타 | 이 선택이 맞을까 | #선택 #결정 |

> 벤치마킹: 점신은 40+ 테마 운영. MVP는 11개로 시작, 점진 확장.

#### 2. 카드 뽑기 (쓰리카드 고정)
- MVP는 쓰리카드(과거/현재/미래) 고정 — 스프레드 선택 없음
- 카드 22장 뒷면 그리드 배치 (메이저 아르카나)
- 상단에 결과 슬롯 3개 미리 표시 (과거/현재/미래)
- 터치로 선택 → 뒤집기 애니메이션
- 정방향/역방향 랜덤 결정
- 3장 선택 완료 시: "다시 선택" / "결과 보기" 버튼

#### 3. 광고 시청 (결과 게이트)
- 결과 보기 전 전면 광고(Interstitial) 매회 1회 노출
- 웹: Google AdSense Interstitial / 앱: AdMob Interstitial (Capacitor 플러그인)
- 기회 제한 없음 — 매번 무료 이용 + 매번 광고
- 결과 화면 하단에 배너 광고 1개 추가 배치 (홈 화면 배너 없음)

#### 4. AI 해석 결과
- 선택한 카드 이미지 + 카드명
- 정방향/역방향 표시
- AI 생성 해석 문장 (스트리밍 표시) — 점신 대비 차별점
- 공유 버튼, 저장 버튼

#### 5. 공유
- "공유하기" 클릭 시 결과를 Supabase DB에 저장 → shareId 생성
- 공유 URL: `/shared/:shareId` (깔끔한 URL)
- OG 태그: Supabase Edge Function으로 동적 생성 (카드 이미지, 해석 요약)
- MVP: 링크 복사. 카카오톡/인스타 스토리 → P0 백로그
- 공유 결과 페이지 하단 CTA: "나도 점 하나 찍어볼까?" → 홈 이동
- DB 보존 기간: 30일 후 자동 삭제 (용량 관리)

#### 6. 히스토리
- 이전 해석 결과 목록 (최신순, 최대 20건)
- localStorage 기반 (서버 동기화는 P0 백로그)
- 빈 상태 안내: "여기는 아직 비어있어요. 점 하나 찍으러 가볼까요?" + 홈 이동 버튼
- 항목 탭 → 결과 페이지 이동
- 미완료 세션은 표시하지 않음

## 에지 케이스

### AI 서버 다운 시

```
결과 보기 클릭
  → Ollama 호출 (타임아웃 30초)
  → 실패 시 조용히 1회 재시도 (타임아웃 15초)
  → 재실패 시:
      1. 미완료 세션을 localStorage에 저장
      2. 오류 모달 표시:
         "일시적으로 해석을 불러오지 못했어요.
          잠시 후 다시 시도해주세요."
         [홈으로 돌아가기]
```

**미완료 세션 복구:**
- 홈 진입 시 미완료 세션 감지 → 모달 표시
- 모달 내용:
  - "확인하지 못한 결과가 있어요"
  - [이어서 보기] — 광고 스킵 → 바로 AI 해석 요청
  - [닫기] — 모달 닫힘. 다음 홈 진입 시 다시 표시
  - □ 더 이상 보지 않기 — 체크 시 당일 자정까지 모달 억제
- 2~3건인 경우 모달 내 목록 표시

**미완료 세션 관리 정책:**
- 최대 3건 (FIFO — 3건 초과 시 가장 오래된 것 삭제)
- 만료: 당일 자정(00:00) 자동 삭제
- 이어서 보기 중 AI 또 실패 시 → 동일 플로우 반복 (세션 유지)
- 미완료 세션과 새 점 시작은 독립 (새 점 정상 진행 가능)

**모달 억제 플래그:**
- 미완료 세션 데이터와 독립 관리
- "더 이상 보지 않기" 체크 시 당일 자정까지 모달 억제
- 억제 중 신규 미완료 세션 발생해도 모달 안 뜸
- 억제 상태에서 미완료 세션 접근 경로 없음 (의도적)
- 자정에 억제 플래그 + 미완료 세션 모두 자연 만료
- 히스토리 페이지에서 미완료 세션 표시하지 않음

### 광고 로드 실패 시

### 카드 뽑기 도중 뒤로가기
- 확인 다이얼로그 없이 즉시 홈 이동
- 선택 상태 초기화

### 같은 카드 중복 선택 방지
- 선택된 카드는 그리드에서 반투명 처리
- 재탭 시 아무 반응 없음 (탭 무시)
- 별도 토스트/피드백 없음

### 네트워크 오프라인
- MVP에서 별도 처리 없음. A-4-1(AI 다운), A-4-2(광고 실패) 에러 처리로 커버
- 오프라인 전용 UI는 P1 백로그

### 광고 로드 실패 시

```
결과 보기 클릭
  → 공통 전체 페이지 로딩 화면 (터치/뒤로가기 차단)
  → 광고 로드 시도 (타임아웃 5초)
  → 성공 → 로딩 해제 → 광고 시청 → 결과 페이지
  → 실패 → 로딩 해제 → 조용히 스킵 → 바로 결과 페이지
```

- 사용자에게 실패를 알리지 않음 (조용히 스킵)
- GA4 이벤트 `ad_failed`로 실패율 추적

### 공통 전체 페이지 로딩 화면

- 문구 없음, 스피너 또는 로고 애니메이션만
- 터치 / 뒤로가기 차단
- 광고 로드, AI 호출 등 비동기 대기에 공통 사용

## localStorage 정책

### 저장 대상
- 결과 히스토리: 최대 20건
- 미완료 세션: 최대 3건 (당일 자정 만료)

### 암호화 + 압축
모든 localStorage 데이터에 암호화 + 압축 적용.
- 처리 순서: `JSON → lz-string 압축 → AES 암호화 → base64 → localStorage`
- 읽기 순서: `localStorage → base64 → AES 복호화 → lz-string 해제 → JSON`
- 목적: 개발자 도구 평문 노출 방지 + 용량 절감

### DB 스키마 (MVP)

MVP에서 DB는 공유 기능 전용. 결과 히스토리는 localStorage.

```sql
-- 공유 결과 저장 (공유하기 클릭 시에만 INSERT)
CREATE TABLE shared_readings (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  theme_id    text NOT NULL,
  theme_title text NOT NULL,
  cards       jsonb NOT NULL,        -- [{cardId, positionIndex, isReversed}]
  interpretation text NOT NULL,      -- AI 해석 전문
  summary     text NOT NULL,         -- 공유용 한줄 요약
  created_at  timestamptz DEFAULT now(),
  expires_at  timestamptz DEFAULT now() + interval '30 days'
);

CREATE INDEX idx_shared_readings_expires ON shared_readings (expires_at);

-- RLS
ALTER TABLE shared_readings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "anyone can read" ON shared_readings FOR SELECT USING (true);
CREATE POLICY "anyone can insert" ON shared_readings FOR INSERT WITH CHECK (true);
```

- 만료 삭제: pg_cron 또는 Supabase scheduled function (`DELETE FROM shared_readings WHERE expires_at < now()`)
- readings 테이블 (유저별 결과 서버 저장): P0 백로그 — 회원가입 구현 후 추가

## 아키텍처

> 벤치마킹 반영: 기회 관리 제거, 매회 광고 시청 모델로 전환. Edge Function 1개로 축소.

```
[앱 (클라이언트)]
  ├── 테마 목록 / 카드 뽑기 (랜덤) ────→ 자체 처리 (정적 데이터)
  ├── AI 해석 요청 ─────────────────────→ Edge Function (interpret)
  │                                         ├── 프롬프트 조합
  │                                         ├── Ollama 호출 (Cloudflare Tunnel)
  │                                         └── SSE 스트리밍 중계 → 클라이언트
  ├── 결과 저장/조회 ──────────────────→ Supabase DB (SDK + RLS)
  └── 광고 표시 ────────────────────────→ Google AdMob/AdSense (클라이언트)
```

## API 설계

### Edge Functions (서버 검증 필수 — MVP 1개)

#### 1. 해석 요청 + AI 스트리밍 (interpret)
```
POST /functions/v1/interpret
```
- 요청: `{ themeId: string, cards: DrawnCard[] }`
- 처리:
  1. 카드 정적 데이터 + 테마 정보 + 스프레드 위치로 프롬프트 조합
  2. Ollama API 호출 (`OLLAMA_URL` — Supabase Secret, 백엔드 전용)
  3. SSE 스트리밍 응답을 클라이언트로 중계
- 응답 (200): SSE 스트리밍 (해석 문장)

## 상태 관리

### Zustand 스토어

| 스토어 | 역할 | 영속성 |
|---|---|---|
| `useReadingStore` | 현재 뽑기 세션 (선택된 카드, 테마ID, 진행 상태) | 메모리만 (페이지 이탈 시 초기화) |
| `useHistoryStore` | 완료된 결과 목록 (최대 20건) | localStorage (암호화+압축) |
| `usePendingStore` | 미완료 세션 (최대 3건, 당일 자정 만료) | localStorage (암호화+압축) |

- 스토어 분리 기준: 영속성 (메모리 vs localStorage)
- 카테고리 필터: URL 쿼리 파라미터로 관리, 스토어 불필요

### localStorage 관리

- **스토리지 유틸 단일 모듈** (`storageUtil`): 암호화(AES) + 압축(lz-string) + 읽기/쓰기를 하나로 통합
- Zustand `persist` 미들웨어: 커스텀 storage로 스토리지 유틸 사용
- 모달 억제 플래그 (`suppressedUntil`): 스토어 밖에서 동일한 스토리지 유틸로 직접 관리 (미완료 세션과 독립, 당일 자정 만료)
- 모든 localStorage 접근은 스토리지 유틸 경유 — 직접 `localStorage.getItem/setItem` 호출 금지

### 카드 선택 경험

> 카드 뽑기는 서비스의 핵심 경험. "내가 고른 카드"라는 감각을 만드는 것이 목표.
> 세부 타이밍/이징은 스케치 및 구현 시 보정.

**1단계 — 고르기 전 (기대감)**
- 카드에 손이 가까워지면(호버/터치) 카드가 살짝 떠오름
- 미세한 빛남 효과 — "이 카드가 나를 부르는 것 같은" 느낌

**2단계 — 선택 순간 (몰입)**
- 터치 시 카드가 위로 떠오르며 뒤집힘
- 단순 회전이 아닌, 공중에서 천천히 열리는 느낌
- 뒤집히는 동안 주변 카드가 살짝 어두워짐 (선택된 카드에 집중)
- 시각 피드백이 메인: 카드 선택 시 짧은 스케일 펄스(1.05x → 1.0x) + 주변 어두워짐
- 햅틱 피드백: 모바일 앱(Capacitor) 햅틱, 모바일 웹 Vibration API (미지원 시 무시), 데스크톱은 시각만
- 진동 기본 ON, 토글은 설정 페이지(백로그)에서 제공 예정. MVP는 기기 설정으로 대체

**3단계 — 공개 (설렘)**
- 카드 앞면이 보이면서 부드러운 글로우
- 카드명이 아래에서 페이드인
- 정방향: 카테고리 포인트 색상 글로우
- 역방향: 카드가 거꾸로(180도) 공개 + 부드러운 보라 글로우 + "역방향" 라벨 페이드인

**4단계 — 자리잡기 (완성감)**
- 카드가 상단 슬롯으로 부드럽게 이동 (스프링 커브)
- 슬롯에 안착하면서 가벼운 바운스
- "과거" / "현재" / "미래" 라벨이 함께 활성화

**구현 방향:** Framer Motion (스프링/제스처 지원)

### 카드 이미지 에셋

**방식: AI 생성 커스텀 (Midjourney/DALL-E)**
- 메이저 아르카나 22장 + 카드 뒷면 1장 = 총 23장
- "밝고 귀여운" 브랜드 톤에 맞춘 일러스트 스타일
- 22장 일관성 유지를 위해 동일 프롬프트 베이스 + 카드별 변수만 변경
- 이미지 규격: 스케치에서 확정 (권장 비율 2:3, WebP 포맷)
- 카드 뒷면: 점하나 로고/심볼 + 카테고리 무관 공통 디자인

**생성 가이드 키워드 (방향):**
- 스타일: flat illustration / cute / pastel / minimal
- 톤: 밝고 따뜻한, 전통 타로 클리셰 배제
- 각 카드의 전통적 상징을 귀여운 일러스트로 재해석
