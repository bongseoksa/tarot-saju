## D-2. 상태 관리 설계

- **상태**: [x] 정의 완료
- **결정**:
  - `useReadingStore`: 현재 뽑기 세션 (메모리만, 이탈 시 초기화)
  - `useHistoryStore`: 완료 결과 목록 최대 20건 (localStorage, 암호화+압축, Zustand persist)
  - `usePendingStore`: 미완료 세션 최대 3건, 당일 자정 만료 (localStorage, 암호화+압축, Zustand persist)
  - 모달 억제 플래그(`suppressedUntil`): 스토어 밖에서 스토리지 유틸로 직접 관리 (독립)

  - **스토리지 유틸 단일 모듈** (`storageUtil`): 모든 localStorage 접근을 하나의 유틸(암호화+압축+읽기/쓰기)로 통합
