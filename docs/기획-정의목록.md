- **상태**: [x] 정의 완료
- **결정**:
  - `/` — 홈 (SEO 프리렌더 대상)
  - `/reading/:themeId` — 카드 뽑기
  - `/result/:resultId` — 결과 (resultId = 로컬 UUID)
  - `/history` — 히스토리 목록
  - `/shared/:shareId` — 공유 결과 (D-3에서 상세 확정)
  - 404: 캐치올 → 홈 리다이렉트 (별도 페이지 없음)
  - 약관/개인정보: 노션 외부 링크, 라우트 불필요
  - 광고 게이트: 별도 URL 없음, 오버레이 처리
  - 카테고리 필터: `/?category=love` (선택적)
- **기록 위치**: docs/specs/02-taro-mvp.md (라우팅 섹션)
- **선행**: A-3

## D-2. 상태 관리 설계

- **상태**: [x] 정의 완료
- **결정**:
  - `useReadingStore`: 현재 뽑기 세션 (메모리만, 이탈 시 초기화)
  - `useHistoryStore`: 완료 결과 목록 최대 20건 (localStorage, 암호화+압축, Zustand persist)
  - `usePendingStore`: 미완료 세션 최대 3건, 당일 자정 만료 (localStorage, 암호화+압축, Zustand persist)
  - 모달 억제 플래그(`suppressedUntil`): 스토어 밖에서 스토리지 유틸로 직접 관리 (독립)
  - 카테고리 필터: URL 쿼리 파라미터 (D-1), 스토어 불필요
  - 분리 기준: 영속성 (메모리 vs localStorage)
  - **스토리지 유틸 단일 모듈** (`storageUtil`): 모든 localStorage 접근을 하나의 유틸(암호화+압축+읽기/쓰기)로 통합
- **기록 위치**: docs/specs/02-taro-mvp.md (상태 관리 섹션)
- **선행**: A-5

- **상태**: [x] 정의 완료

  - URL: `/shared/:shareId` — 깔끔한 공유 URL
  - OG 태그: Supabase Edge Function으로 동적 생성 (카드 이미지, 해석 요약)
  - 공유 페이지 CTA: "나도 점 하나 찍어볼까?" → 홈 이동
  - 보존 기간: 30일 후 자동 삭제 (DB 용량 관리)
